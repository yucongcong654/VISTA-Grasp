# 行为准则 (Code of Conduct)

## 核心原则

VISTA-Grasp 团队致力于为所有开发者、贡献者和最终用户提供一个开放、包容、友善和无骚扰的社区互动环境。

## 核心承诺

我们承诺，无论参与者的年龄、性别、性别认同与表达、性取向、残障情况、种族、民族、宗教信仰、地域、社会经济地位或技术水平如何，都将为其提供友好、尊重、无歧视的参与环境。我们致力于打造一个无骚扰、无欺凌、无歧视的社区，让每一位参与者都能安心贡献、平等交流。只要您热衷于具身智能与开源技术，我们都非常欢迎您的参与。我们坚信多元化的社区才能迸发出最强的技术生命力。


## 社区期望行为

我们期望并鼓励所有社区成员在参与讨论、提交 Issue 和 Code Review 时，主动遵守以下正面行为规范：

- **友善与包容**：使用包容性的语言，尊重他人的观点、经验和不同的技术选择。
- **建设性交流**：以推动项目进步和解决实际问题为核心目标。优雅、坦诚地接受他人的意见，同时也为他人提供有建设性、带有指导意义的反馈。
- **聚焦项目本身**：保持所有的技术讨论集中在项目的代码、架构规划、复现 Bug 和社区建设上。
- **乐于助人**：在您的能力范围内，友善地帮助那些刚刚接触项目的新手解答问题。

## 不可接受的违规行为

社区对于以下违规行为采取零容忍态度，包括但不限于：

- 使用带有性暗示的语言、图像或引起他人极度不适的过分关注。
- 挑衅性言论、侮辱、贬损的评论，以及针对任何个人的直接人身攻击。
- 在 Issue、PR 或社群中公开或私下骚扰他人。
- 未经明确许可，公开发布他人的私人隐私信息（如家庭地址、私人电子邮箱等）。
- 发布、传播包含敏感、违法违规或具有破坏性的内容。
- 抄袭他人代码、文档或创意，未注明来源即作为自己的贡献提交。
- 其他任何在专业职场环境中被视为不当或不道德的行为。

## 贡献规范

- 代码贡献：提交代码前需进行充分测试，确保代码无明显bug、符合项目编码规范，提交PR时需详细说明修改内容、目的及测试情况。
- 文档贡献：完善项目文档（包括使用手册、API文档、开发指南等），确保内容准确、清晰、易懂，中英文文档需保持同步更新。
- 反馈与建议：提交issue时，需明确描述问题、复现步骤（如有），避免模糊不清的表述；提出建议时，需结合项目实际需求，给出具体可行的方案。
- 社区讨论：在社区交流中，聚焦技术问题和项目发展，避免发表无关言论、传播谣言或引发不必要的争议。

## 开源协议说明
本项目采用Apache-2.0，所有贡献者提交的代码、文档等内容，将自动遵循本项目的开源协议，贡献者保留自身著作权，同时授予项目维护者及社区合理使用、修改、分发的权利。详情请参考项目根目录下的LICENSE文件。

本公司仅作为项目的维护者提供此交流平台，对于社区个人用户的违规言论和非法行为不承担任何法律连带责任。
